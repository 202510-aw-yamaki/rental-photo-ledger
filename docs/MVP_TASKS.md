# MVP_TASKS.md

## 目的

このファイルは、Codex appにMVPを小さな単位で作業してもらうためのタスク一覧です。

一度にすべてを作らず、上から順番に進めます。

## Task 0: プロジェクト土台作成

### 目的

React + TypeScript + Vite + Tailwind CSS のPWAアプリとして開発できる土台を作る。

### 完了条件

- React + TypeScript + Vite の構成になっている
- Tailwind CSS が使える
- 開発サーバーが起動できる
- トップ画面が表示される
- アプリ名が表示される
- 法律判断をしない旨の注意文が表示される

### 表示する文言

アプリ名:

```text
賃貸写真台帳メーカー
```

説明文:

```text
入居時・退去時の部屋の状態を、間取り図・写真・コメントで整理して残すためのアプリです。
```

注意文:

```text
このアプリは法律判断を行いません。部屋の状態を記録・整理することを目的としています。
```

## Task 1: データ設計

### 目的

MVPで必要なデータ型と保存処理を作る。

### 完了条件

- TypeScriptの型定義がある
- IndexedDB保存の方針が決まっている
- Dexieを使ったDB層がある
- 基本CRUDがある
- サンプルデータ作成関数がある

### 必要データ

- Property
- FloorPlan
- FloorPlanPin
- PhotoRecord
- ChecklistItem
- ExportHistory

## Task 2: 物件管理

### 目的

物件を作成し、一覧と詳細を表示できるようにする。

### 完了条件

- 物件一覧画面がある
- 新規物件作成画面がある
- 物件詳細画面がある
- 物件名を保存できる
- 入居日を保存できる
- 退去日を任意で保存できる

## Task 3: 間取りPDFアップロード

### 目的

物件詳細から間取りPDFをアップロードして保存・表示できるようにする。

### 完了条件

- PDFを選択できる
- PDFを保存できる
- PDFの1ページ目を表示できる
- エラー時に日本語メッセージを表示する
- スマートフォンでも見やすい

## Task 4: 番号ピン配置

### 目的

間取りPDFの上に番号ピンを置けるようにする。

### 完了条件

- PDF上をクリックまたはタップしてピンを追加できる
- ピン番号が連番になる
- ピンに場所名を入力できる
- ピンを移動できる
- ピン座標を保存できる
- ピン座標は正規化座標で保存する

## Task 5: 写真登録

### 目的

番号ピンごとに写真を登録できるようにする。

### 完了条件

- ピン詳細画面がある
- 写真を追加できる
- スマートフォンのカメラまたは写真選択に対応する
- 写真ごとにカテゴリーを選べる
- 写真ごとにコメントを入力できる
- 登録日時を保存する
- 写真一覧を表示する

## Task 6: 入居時チェックリスト

### 目的

撮影漏れを減らすためのチェックリストを表示・保存できるようにする。

### 完了条件

- チェックリスト画面がある
- 項目をチェック済みにできる
- 項目ごとにメモを残せる
- 初期項目が用意されている

## Task 7: PDF台帳出力

### 目的

記録した内容をPDF台帳として保存できるようにする。

### 完了条件

- PDFを生成できる
- 表紙がある
- 間取り図ページがある
- 写真台帳ページがある
- 日付が入る
- 注意書きが入る
- 日本語が文字化けしない
- スマートフォンで保存できる

## Task 8: 共有

### 目的

作成した台帳を共有しやすくする。

### 完了条件

- Web Share APIが使える場合は共有ボタンを出す
- 使えない場合はPDF保存ボタンを出す
- 共有用メール文面をコピーできる
- 法律判断に見える表現を避ける
